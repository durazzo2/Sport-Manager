3. Database Schema
Create migrations using UUIDs as primary keys.

User: Auth fields, phone, role (via Shield).

Amenity: name (e.g., "Parking", "Wifi"), icon (Heroicon string).

Rental: name (e.g., "Racket", "Ball"), price (integer/cents).

Facility: name, description, city (indexed), address. Has Many Courts, Belongs to Many Amenities.

Court: facility_id (FK), name, type (enum: 'Football', 'Tennis', 'Padel', 'Swimming'), base_price_per_hour (integer/cents), image_path.

Booking:

user_id (FK), court_id (FK).

start_time (datetime), end_time (datetime).

status (enum: 'confirmed', 'cancelled', 'completed', 'checked_in').

total_price (integer/cents) - Stores final price after dynamic rules.

qr_code (string/text) - Stores the unique check-in string.

Constraint: Prevent overlapping bookings on the DB level.

BookingRental (Pivot): Links Booking and Rental (with quantity).

Review: user_id, facility_id, rating (1-5), comment.
